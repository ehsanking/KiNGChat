# تحقیق منابع موردنیاز برای کپچای داخلی (بدون سرویس خارجی)

این سند بر اساس README اصلی تهیه شده و جمع‌بندی می‌کند برای راه‌اندازی کپچای بدون وابستگی خارجی چه منابعی لازم است.

## منابع ضروری

1. **منطق تولید چالش در سرور**
   - تولید یک چالش ساده (مثل جمع دو عدد) به همراه `captchaId` امضاشده.
   - فایل: `lib/local-captcha.ts`

2. **اعتبارسنجی استیت‌لس در سرور**
   - بررسی زمان انقضا و امضای HMAC بدون نیاز به Redis یا دیتابیس اختصاصی.
   - فایل: `lib/local-captcha.ts`

3. **API داخلی برای گرفتن چالش**
   - endpoint محلی: `GET /api/captcha`
   - فایل: `app/api/captcha/route.ts`

4. **کامپوننت فرانت برای نمایش و دریافت پاسخ**
   - نمایش عبارت ریاضی + ورودی پاسخ + دکمه رفرش.
   - فایل: `components/LocalCaptcha.tsx`

5. **اتصال به جریان‌های احراز هویت**
   - در login/register، هنگام فعال بودن captcha پاسخ کاربر بررسی شود.
   - فایل: `app/actions/auth.ts` و صفحات auth

## منابع اختیاری ولی توصیه‌شده

- `CAPTCHA_SECRET` با طول حداقل ۱۶ کاراکتر برای امضای پایدار بین ری‌استارت‌ها.
- Rate limit روی endpointهای login/register/captcha.

## مزیت معماری پیشنهادی

- بدون سرویس خارجی
- بدون SDK خارجی کپچا
- بدون تماس شبکه به provider ثالث
- استقرار ساده‌تر در محیط‌های آفلاین/داخلی
